## 필수과제에서 구현기능

### 1. 게임 로직 (GameMode)
- `GenerateSecretNumber`: 1~9 사이의 중복 없는 랜덤 숫자 3개를 생성합니다.
- `PrintPlayerInputcalculate` : 플레이어가 입력한 글자 분석후 모든 플레이어에게 로그를 출력합니다. 아래 함수들은 해당 함수에서 이용됩니다.
- `GameStart` : 게임 시작을 위해 정보들을 초기화합니다.
- `CheckPlayerChance` : 해당 플레이어에게 입력할 기회가 잇는지 확인합니다. 게임이 종료거나 기회가 다 소진되었는지 체크합니다.
- `CheckString` : 입력된 글자가 겹치지 않는 세자리 숫자인지 검사합니다.
- `JudgeResult`: 플레이어의 입력을 정답과 비교하여 Strike와 Ball 개수를 판정합니다.
- `ResultMessageMix`: 문자열들과 정보를 조합하여 입력 결과를 출력할 문자열을 만듭니다.
- `IncreaseCount` : 시도 횟수를 증가시킵니다.
- `JudgeGame`: 승리 조건(3 Strike) 또는 무승부 조건(모든 기회 소진)을 체크합니다.
- `GameEnd` : 게임종료시 사용되며 결과를 발표하고 게임 종료상태가 됩니다. 
- `ReStartGame`: 게임 종료 후 일정 시간(5초) 뒤 상태를 초기화하고 정답을 재생성합니다.

### 2. 플레이어 조종 (PlayerController)
- `PrintChat` : 입력된 채팅을 서버로 보냅니다.
- `Server RPC`: 클라이언트에서 전달된 입력을 서버로 전달하여 GameMode에서 검증 및 계산을 수행합니다.
- `Client RPC`: GameMode에서 계산된 결과 메시지를 받습니다.
- `ChatInputText`: 결과 메세지를 출력하도록 합니다.  

### 3. 데이터 관리 (PlayerState)
- 플레이어의 고유 이름(예: 춘식이_0), 현재 시도 횟수, 최대 시도 횟수 등을 관리합니다.
- 레플리케이션을 사용하여 모든 플레이어의 상태 정보를 서버와 클라이언트 간 동기화합니다.

### 4. 사용자 인터페이스 (UI)
- TextInput: EditableTextBox를 통해 숫자를 입력받고, Enter 키 입력 시 컨트롤러에게 입력된 텍스쳐를 전송합니다.
- MainLog: 모든 게임 진행 상황(입력 값, 판정 결과, 승리 알림 등)을 스크롤 박스에 텍스트 박스를 추가하여 로그를 표시합니다.

###  규칙
1.  게임 실행 후 하단의 입력창에 **중복 없는 숫자 3자리**를 입력하고 Enter를 누릅니다.
2.  결과는 `[이름] : 입력값 -> 결과: XSYB (시도횟수: N/M)` 형태로 로그에 표시됩니다.
3.  **3 Strike**를 먼저 달성하는 플레이어가 승리합니다.
4.  지정된 시도 횟수(기본 3회) 내에 맞추지 못하면 입력 기회를 잃게 됩니다.
5.  게임이 종료되면 5초 후 자동으로 새로운 정답과 함께 게임이 다시 시작됩니다.
